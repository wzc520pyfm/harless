# harless v0.1 ÔÇö Design Spec

- **Date:** 2026-04-20
- **Status:** Approved (brainstorming complete)
- **Author:** harless project
- **Audience:** future implementers, contributors, and adopters

---

## 1. Project Statement

`harless` is an **agent-agnostic, minimal AI Coding Harness kit**. Through `npx harless init` / `add`, it injects 8 capability modules into a user's project as plain markdown files and shell scripts. Any AI agent that respects the [`AGENTS.md`](https://agents.md) convention (Claude Code, Cursor, Codex CLI, Copilot CLI, Gemini CLI, ÔÇŽ) gains an immediate, complete production loop:

> Skills ┬Ě Spec ┬Ě Loop ┬Ě Memory ┬Ě Browser-Debug ┬Ě Orchestration ┬Ě Review ┬Ě Simplify

The goal is not to replace heavy alternatives (Superpowers, OpenSpec, claude-mem, agentation, etc.) but to provide the **smallest possible complete subset** that global Web developers can adopt at near-zero cost, and that is easy to learn, extend, and fork.

### 1.1 Non-goals (v0.1)

- Replacing Superpowers / OpenSpec / claude-mem feature-by-feature
- Self-built MCP server (deferred to v0.2+)
- Slash commands, hooks, plugins, subagent-specific configurations
- SaaS backend, telemetry, accounts, online dashboards
- Multi-OS matrix beyond macOS + Linux (Windows deferred to v0.2)
- A spec validator (deferred to v0.2+)

## 2. Locked Architectural Decisions

| # | Decision | Rationale |
|---|---|---|
| D1 | **Agent-agnostic** via `AGENTS.md` + Read tool + non-interactive CLI | Avoids per-agent adapter sprawl; serves global developers |
| D2 | **CLI-injected files (shadcn/OpenSpec style)** via `npx harless init` / `add` | Code lives in user's repo; trivial fork & customize |
| D3 | **Two primitives only**: Markdown + Shell/Node scripts | Keeps the surface tiny and learnable |
| D4 | External MCP only ÔÇö reuse `chrome-devtools-mcp`, no self-built MCP | YAGNI; community already maintains it |
| D5 | **Web-first defaults + generic degradation** | Auto-detect Next/Vite/React; degrade via `--stack=generic` |
| D6 | **Smoke-tested on Tier 1 = CC, Cursor, Codex CLI** | Three formats represent the field; others are AGENTS.md-compatible |
| D7 | **No daemons, no hooks, no global config writes** | Zero side effects on user's machine |
| D8 | Project name = **`harless`** | Short, unique, self-evidently "harness minus weight" |

### 2.1 Five Design Axioms (Inviolable)

1. **Zero runtime** ÔÇö no daemons, no port listeners, no background workers.
2. **Two primitives** ÔÇö only Markdown files and shell/Node scripts; external MCP only via reuse.
3. **Code belongs to the user** ÔÇö files written by `add` are user-owned; `update` uses a 3-way diff merge, never a clobber.
4. **AGENTS.md is the protocol** ÔÇö all cross-agent compatibility flows through one file at the project root.
5. **YAGNI ruthlessly** ÔÇö no slash commands, no self-MCP, no plugins, no GUI, no SaaS in v0.1.

## 4. CLI Surface

### 4.1 Commands

| Command | Purpose | Idempotent |
|---|---|---|
| `harless init` | Detect stack, interactively select modules, write files | Ôťů |
| `harless add <module>` | Add one module after init | Ôťů |
| `harless remove <module>` | Remove one module (with hash-aware safety) | Ôťů |
| `harless update` | Hash-based update with `[k]eep / [o]verwrite` per file | Ôťů |
| `harless doctor` | Inspect environment, agent config, MCP, permissions | Ôťů (read-only) |

**Global flags (apply to all commands):**

- `--yes` ÔÇö non-interactive; semantics per ┬ž4.3.
- `--offline` ÔÇö skip the optional npm-registry version check and any other network probe (e.g., MCP package metadata lookup). Useful in air-gapped CI. Has no effect on file operations.

**Not in v0.1:** `login`, `publish`, `share`, `ai`, `run`, `watch`, `spec new`, `memory query`. The CLI is **not** a second agent UI ÔÇö agents call scripts and read files directly.

### 4.2 Exit-Code Convention (consolidated)

| Exit | Meaning | Used by |
|---|---|---|
| `0` | success | all commands |
| `1` | unrecoverable error (filesystem, network, malformed config) | all commands |
| `2` | unresolved conflict / user input required but `--yes` was set | `init`, `add`, `update`, `remove` |
| `3` | precondition failed (Node version, missing dependency) | all commands |
| `10`+ | `doctor` only: bitmask of failing checks (10=AGENTS, 11=MCP, 12=perms, ÔÇŽ) | `doctor` |

### 4.3 `--yes` Semantics per Command

| Command | With `--yes` | Without |
|---|---|---|
| `init` | use defaults; on AGENTS.md/MCP merge conflict Ôćĺ exit 2 | interactive prompts |
| `add <m>` | overwrite-if-pristine, exit 2 if user-modified | prompt `[k]eep / [o]verwrite` |
| `update` | overwrite-if-pristine, exit 2 if conflict | prompt `[k]eep / [o]verwrite` |
| `remove <m>` | delete pristine files; move user-modified files to `.harness/.trash/<ts>/` (per ┬ž4.6) | per-file `[d]elete / [t]rash / [k]eep` prompt (┬ž4.6) |
| `doctor` | n/a (read-only) | n/a |

### 4.4 `init` Behavior

1. Detect stack (`package.json` heuristics: `next`, `vite`, `react`, `remix`, `astro`, `svelte`).
2. Detect package manager (`pnpm-lock.yaml` / `yarn.lock` / `package-lock.json`).
3. **Monorepo handling**: if a workspaces field is detected (`pnpm-workspace.yaml`, `package.json#workspaces`, `lerna.json`), `init` runs at the **workspace root**; warns and exits 1 if invoked from inside a sub-package (with override flag `--here`).
4. Interactive multi-select: 6 modules pre-checked (`skills, spec, loop, memory, browser-debug, review`), 2 unchecked (`orchestrate, simplify`).
5. Interactive multi-select: smoke-tested agent (CC, Cursor, Codex CLI, Other).
6. Write files; merge `AGENTS.md` and `.mcp.json` if present.
7. Print "Next steps" with verification command and a `.gitignore` snippet suggestion (see ┬ž13.3).

**Non-interactive form:** `--yes --modules=... --agents=...` for CI.

### 4.5 `update` ÔÇö Hash-Based Per-File Action (v0.1: keep / overwrite only)

Each templated file's SHA-256 is recorded in `.harness/config.json` at install time. On `update`:

| Disk hash vs old template | New template vs old template | v0.1 action |
|---|---|---|
| same | changed | silent overwrite |
| changed | same | leave disk untouched |
| changed | changed | print unified diff; user picks `[k]eep / [o]verwrite` |
| missing | n/a | offer to recreate |

**`[m]erge` (3-way text merge via `git merge-file`) is deferred to v0.2.** v0.1 deliberately ships only the two safe options to keep the CLI ÔëĄ 500 LOC and the conflict UX trivially understandable. Users who want a real merge can manually run `git merge-file` themselves; the diff output prints both sides verbatim.

**No automatic clobber on conflict, ever.**

### 4.6 `remove` Behavior

For each file the module owns:
1. If file's hash matches recorded template hash Ôćĺ delete (file was never edited).
2. If file's hash differs Ôćĺ file is user-modified:
   - Without `--yes`: prompt `[d]elete / [t]rash / [k]eep` (default trash).
   - With `--yes`: move to `.harness/.trash/<YYYY-MM-DD-HH-MM-SS>/` (preserving relative path), then update `config.json`.
3. Always remove the corresponding entries from `AGENTS.md`'s Skills Index table and the module's section in `config.json`.

Trash is never auto-purged in v0.1; user can `rm -rf` when ready.

### 4.7 `doctor` Checks (v0.1: snippet-printer, not runtime probe)

- Node Ôëą 18 (probed)
- `AGENTS.md` exists at root and contains the `harless` managed block (probed)
- `.harness/scripts/*.sh` are executable (probed)
- If `browser-debug` enabled: `.mcp.json` lists `chrome-devtools-mcp` (probed); **Chrome reachability is NOT probed** in v0.1 ÔÇö too platform-specific. Doctor only prints the install command.
- For each declared agent: print the **paste-ready snippet** for that agent's MCP config path (Codex `~/.codex/config.toml`, Gemini `.gemini/settings.json`, Copilot config). **Doctor does not write or read those files** ÔÇö privacy and YAGNI.

Each check returns Ôťô / ÔÜá / ÔťŚ with a single-line remediation. Bitmask in exit code (see ┬ž4.2).

## 6. Eight Modules

Each module is **1 SKILL.md + (optional) 1 script**, ÔëĄ 100 LOC markdown and ÔëĄ 80 LOC script (per ┬ž5.2 budget table). The `skills` module is a 5-file bundle; all other modules ship a single SKILL.md.

### 6.1 `skills` ÔÇö discipline bundle (Superpowers minimal subset)

This module is a **bundle of 5 sub-discipline SKILL files** rather than a single SKILL.md. Selected for "agent wouldn't think of it + biggest quality impact":

- `.harness/skills/brainstorming/SKILL.md` (intent before code)
- `.harness/skills/tdd/SKILL.md` (test-first discipline)
- `.harness/skills/systematic-debugging/SKILL.md` (no random guesses)
- `.harness/skills/verification-before-completion/SKILL.md` (evidence over assertions)
- `.harness/skills/receiving-review/SKILL.md` (structured response to review feedback)

Each ÔëĄ 100 LOC. AGENTS.md exposes them under "Disciplines" (always-on) rather than "Capability Modules" (on-demand) ÔÇö see ┬ž5.3. No script.

### 6.2 `spec` ÔÇö Spec-Driven Development (OpenSpec minimal)

Three-step workflow: idea Ôćĺ `spec.md` (what) Ôćĺ `plan.md` (how) Ôćĺ `tasks.md` (atomic tasks). Templates only; no validator in v0.1. Agent copies `.template/` to a dated change folder. Single `SKILL.md` at `.harness/spec/SKILL.md`.

### 6.3 `loop` ÔÇö Ralph self-correcting loop

`.harness/loop/.template/` ships with a 5-line `goal.md` skeleton and a 3-line `check.sh` (`#!/usr/bin/env bash` + a stub). `loop/SKILL.md` instructs the agent to `cp -r .template/ <task>/`, fill in goal+check, then run the script.

`scripts/loop.sh` (ÔëĄ 40 LOC bash):

```bash
#!/usr/bin/env bash
# AGENT_CMD examples:
#   AGENT_CMD="claude -p"          (default)
#   AGENT_CMD="cursor-agent -p"
#   AGENT_CMD="codex exec"
set -euo pipefail
WORK="${1:?usage: loop.sh <work-dir>}"; cd "$WORK"
MAX="${MAX_ITER:-20}"
AGENT_CMD="${AGENT_CMD:-claude -p}"
[[ -f goal.md && -f check.sh ]] || { echo "need goal.md + check.sh in $WORK"; exit 3; }
for i in $(seq 1 "$MAX"); do
  echo "=== iter $i ==="
  $AGENT_CMD "$(cat goal.md)" | tee "iter-$i.log"
  if bash check.sh; then echo "Ôťô done @ iter $i"; exit 0; fi
done
echo "ÔťŚ max iterations reached"; exit 1
```

### 6.4 `memory` ÔÇö cross-session memory (claude-mem minimal, zero daemon)

**I/O contract** (resolves the "fresh process can't read transcript" issue):

- The **calling agent** authors the summary in markdown using its in-context conversation. The script does NOT call any LLM.
- **`scripts/compact.sh`** is a thin appender: it reads markdown from **stdin**, prepends a timestamp header, and appends to `.harness/memory/sessions/<YYYY-MM-DD>.md`.
- Topic files in `.harness/memory/topics/<topic>.md` are written by the agent **directly** (using its Write tool); no script involved. The agent picks topic names matching keywords used at session start.
- **Read**: SKILL.md instructs the agent to `ls .harness/memory/topics/` and `grep` for relevant terms at session start.
- **Filename = retrieval key** (`auth.md`, `deploy-netlify.md`, ÔÇŽ).
- **Secret protection**: SKILL.md's redaction checklist runs before any write; README recommends gitignoring `memory/sessions/`.
- **Log rotation** (footgun mitigation): SKILL.md instructs the agent to summarize-and-truncate any `sessions/<date>.md` file exceeding 500 lines into the corresponding topic files, then leave a one-line stub. No automatic rotation in v0.1.

`scripts/compact.sh` (ÔëĄ 25 LOC bash):

```bash
#!/usr/bin/env bash
# Usage:
#   echo "<markdown synopsis>" | bash .harness/scripts/compact.sh
#   bash .harness/scripts/compact.sh < summary.md
set -euo pipefail
DATE=$(date +%Y-%m-%d); TS=$(date +%Y-%m-%dT%H:%M:%S%z)
DIR=.harness/memory/sessions; mkdir -p "$DIR"
FILE="$DIR/$DATE.md"
{ echo; echo "## Session @ $TS"; echo; cat; echo; } >> "$FILE"
echo "Ôťô appended to $FILE"
echo "Reminder: update topic files in .harness/memory/topics/ for any reusable knowledge."
```

### 6.5 `browser-debug` ÔÇö Web debugging via `chrome-devtools-mcp`

- Zero self-built code; we just merge `chrome-devtools-mcp` into `.mcp.json`.
- The skill's value is **discipline**: snapshot before action, scroll-into-view before click, abandon after 4 failed attempts.

### 6.6 `orchestrate` ÔÇö parallel subagent dispatch (agentation minimal)

**Input format**: `tasks.json` ÔÇö array of objects `{ "id": "string", "prompt": "string", "cwd": "string?", "timeoutSec": number? }`. JSON chosen over YAML to avoid an extra dep (per ┬ž11 decision).

**Output**: per-task log at `.harness/orchestrate/<id>-<ts>.log` (stdout+stderr combined). The script **never interleaves output to terminal** ÔÇö it prints a one-line "started" per task, then a final summary table with paths and exit codes. This solves the "N parallel agents shouting into one terminal" problem.

`scripts/dispatch.sh` (ÔëĄ 60 LOC bash, parses JSON inline with 5 lines of node):

```bash
#!/usr/bin/env bash
set -euo pipefail
TASKS="${1:?usage: dispatch.sh <tasks.json>}"
AGENT_CMD="${AGENT_CMD:-claude -p}"
TS=$(date +%Y-%m-%dT%H-%M-%S)
LOG_DIR=.harness/orchestrate; mkdir -p "$LOG_DIR"
node -e '
  const t = JSON.parse(require("fs").readFileSync(process.argv[1], "utf8"));
  for (const x of t) console.log([x.id, x.prompt, x.cwd||".", x.timeoutSec||0].join("\t"));
' "$TASKS" | while IFS=$'\t' read -r ID PROMPT CWD TIMEOUT; do
  LOG="$LOG_DIR/$ID-$TS.log"
  echo "Ôćĺ $ID (cwd=$CWD, log=$LOG)"
  ( cd "$CWD" && timeout "${TIMEOUT}s" $AGENT_CMD "$PROMPT" ) >"$LOG" 2>&1 &
done
wait
echo
# Final summary table schema (always emitted):
#   id <TAB> exit <TAB> log
# Sorted by exit code descending (failures first); exit code of dispatch.sh
# itself is 0 if all tasks exit 0, else 1.
printf "%-20s %-6s %s\n" "id" "exit" "log"
```

v0.1: parallel-only, no DAG. DAGs decompose into multiple sequential dispatches. Default-off in `init`.

### 6.7 `review` ÔÇö independent diff review

`scripts/review.sh` (ÔëĄ 60 LOC bash) spawns a fresh `$AGENT_CMD` process with **only the checklist + diff** (no main-session context bias). Output goes to `.harness/review/<ts>.md` (per ┬ž6.9 rule #3, **not** `.harness/review-<ts>.md`):

```bash
#!/usr/bin/env bash
set -euo pipefail
REF="${1:-HEAD}"
AGENT_CMD="${AGENT_CMD:-claude -p}"
TS=$(date +%Y-%m-%dT%H-%M-%S)
OUT_DIR=.harness/review; mkdir -p "$OUT_DIR"
OUT="$OUT_DIR/$TS.md"
DIFF=$(git diff "$REF")
CHECKLIST=$(cat .harness/review/SKILL.md)
{ echo "$CHECKLIST"; echo; echo "=== DIFF ==="; echo "$DIFF"; } | $AGENT_CMD > "$OUT"
echo "Ôťô wrote $OUT"
```

Independent process (not subagent) keeps it cross-agent.

### 6.8 `simplify` ÔÇö code simplification scan

Pure-skill primarily; `scripts/simplify.sh` is an accelerator following the same independent-agent pattern as `review`. Output goes to `.harness/simplify/<ts>.md`. Default-off in `init` but **the script is always shipped** if the module is added (kept ÔëĄ 30 LOC).

### 6.9 Common Script Conventions

All scripts honor:
1. `$AGENT_CMD` env (default `claude -p`) for cross-agent. Each script's header lists 3 example invocations (CC / Cursor / Codex).
2. `set -euo pipefail` for fail-loud behavior.
3. Output artifacts to `.harness/<module>/<timestamp>.{log,md}` (never to `.harness/<module>-<timestamp>` at the root).
4. Exit codes per ┬ž4.2 (0 success, 1 unrecoverable, 2 conflict-with-`--yes`, 3 precondition).
5. No deps beyond `git`, `bash`, `node`. No `jq`/`yq`/`parallel` ÔÇö small inline alternatives.

### 7.2 Four Differences We Resolve

**Difference 1 ÔÇö MCP config paths.** Write `.mcp.json` (covers CC + Cursor). For Codex/Copilot/Gemini, print a paste-snippet in `.harness/README.md`. **Never modify global config files.**

**Difference 3 ÔÇö Skill discovery vs manual reference.** SKILL.md frontmatter benefits CC/Cursor's native discovery. AGENTS.md's "Skills Index" table benefits agents without native skill systems. Same SKILL.md serves both.

**Difference 4 ÔÇö AGENTS.md coexistence with other tools.** Marker-delimited managed block (`<!-- BEGIN/END harless v0.1 -->`). `update` only touches that block. If openspec/superpowers blocks are detected, `doctor` notes coexistence and links to a reconciliation doc.

Run on each Tier 1 agent: ask `"Please help me turn this idea into a design spec: a todo list app."` Expected: agent reads `.harness/skills/brainstorming/SKILL.md` **before** acting, then follows it. **A failure on any Tier 1 agent blocks release.**

## 9. Error Handling, Testing, Distribution

### 9.1 Error-Handling Principles

1. **Fail loud, fail early** ÔÇö `process.exitCode` in CLI; `set -euo pipefail` in scripts; preconditions in SKILLs.
2. **Never silently mutate** ÔÇö show a path before any write; require explicit choice on conflict; `--yes` exits 2 on conflict.
3. **Recoverable by hand** ÔÇö delete `.harness/` and re-`init` always recovers; everything is plain text in git.

### 9.2 Test Pyramid

**Unit (vitest):** `mergeAgentsMd`, `mergeMcpJson`, `decideUpdateAction` (the v0.1 hash-compare helper that returns `keep | overwrite | conflict | recreate`), `detectStack`. Coverage Ôëą 80% line, Ôëą 70% branch. AGENTS.md rendering uses snapshot tests.

**Integration (`execa` + temp dirs):** 8 mandatory scenarios:
1. Empty dir `init` (default selection)
2. `init` over existing AGENTS.md containing an `openspec` block
3. `init Ôćĺ add orchestrate Ôćĺ remove orchestrate`
4. `init` Ôćĺ user edits one SKILL Ôćĺ `update` (template unchanged) Ôćĺ user edit preserved
5. `init` Ôćĺ user edits one SKILL Ôćĺ `update` (template also changed) Ôćĺ conflict path; `--yes` exits 2
6. Re-`init` (idempotency)
7. `doctor` exit codes across 7 scenarios (all green / missing perms / missing MCP / missing AGENTS block / etc.)
8. `.mcp.json` with other servers + add `chrome-devtools-mcp` (merge correctness)

**E2E (manual, recorded):** per Tier 1 agent, a checklist + screen recording in `docs/smoke/<agent>.md`. All three must be green to release.

**Out of scope:** real agent CI, semantic LLM-output assertions, multi-OS matrix.

## 13. Operating Edge Cases (v0.1 covered behavior)

### 13.1 Monorepo

- `init` is workspace-aware: detects `pnpm-workspace.yaml`, `package.json#workspaces`, `lerna.json`. Runs at the workspace root and warns + exits 1 if invoked from a sub-package (override: `--here`).
- `.harness/` lives **only at the workspace root**; per-package `.harness/` directories are not supported in v0.1 (and are flagged as a warning by `doctor`).

### 13.2 Coexistence with `CLAUDE.md` / `.cursorrules` / Other AI Convention Files

- `init` detects existing `CLAUDE.md`, `.cursorrules`, `.github/copilot-instructions.md`. **Does not modify them.**
- It writes `AGENTS.md` (creating or merging) and prints a one-paragraph note in the "Next steps" output: "AGENTS.md is the cross-agent standard; you may want to add a one-line `@AGENTS.md` reference to your existing convention files."
- `doctor` lists detected legacy files and reminds the user.

### 13.3 `.gitignore` Suggestions

`init` prints (not writes) a suggested `.gitignore` block:

```
# harless: optional, depending on your team's policy
```

### 13.4 Offline / Air-Gapped Behavior

- `npx harless@latest` requires network on first invocation; subsequent invocations use the npm cache (standard npm semantics).
- All injected files are local; no module ever phones home.
- The optional version-check (┬ž9.4) has a 1-second timeout and silently no-ops on failure ÔÇö `init`/`add`/`update` always succeed offline if the package is cached.

### 13.5 `.harness/` Already Exists / Partial Install

- Re-running `init` on a directory that already has a complete `.harness/` is allowed (idempotent).
- Re-running on a partial `.harness/` (e.g., user `rm`-ed half a module): `init` reports "partial install detected" and offers to fill missing files at recorded hashes; user-modified files are left alone.

### 13.6 Memory Log Rotation

Append-only `sessions/<date>.md` is a footgun for long-lived projects. v0.1 mitigation:

- The `memory` SKILL instructs the agent to summarize-and-truncate any `sessions/<date>.md` exceeding 500 lines into `topics/` files, then leave a one-line stub citing the rotation timestamp.
- No automatic rotation in v0.1; users who want hard caps add a cron / git hook themselves.

### 13.7 Network-Restricted CI

`harless doctor --offline` skips the version check and the npm registry probe; useful in air-gapped CI.
